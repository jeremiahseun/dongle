## 2024-05-13 - O(N) operations in directory traversal
**Learning:** Trying to optimize `os.path.exists()` checks by loading an entire directory into memory via `os.listdir()` to perform a set intersection turns an O(1) operation into an O(N) operation. While faster for small directories due to fewer syscalls, it risks causing performance and memory regressions when encountering directories with massive numbers of files.
**Action:** When optimizing path traversal, avoid replacing individual O(1) checks with operations that scale with the size of the directory contents, unless it can be guaranteed the directory size will always be strictly bounded and small.
